1.	Как осуществляется хранение иерархических данных?
  1.1 Adjacency List (Список смежности): Самый простой способ, где каждая запись содержит ссылку на идентификатор своего родителя (Parent_ID).
  1.2 Path Enumeration (Перечисление путей / Materialized Path): В каждой записи хранится полный путь от корня до этого узла в виде строки (например, 1/3/5/22) 
  1.3 Nested Sets (Вложенные множества): Узлы описываются интервалами целых чисел (Left_Key, Right_Key).
  1.4 Closure Table (Таблица связей): Создается отдельная таблица, в которой хранятся все пары «предок — потомок» для каждого уровня глубины. 
 
2.	Какие типичные задачи решаются при хранении иерархических данных?
  2.1 Обход дерева (Traversal): Вывод всего дерева или отдельной ветви в правильном визуальном порядке (с отступами).
  2.2 Поиск корня или листьев: Определение самого верхнего элемента или элементов, не имеющих потомков.
  2.3 Агрегация (Roll-up): Подсчет суммарных значений по ветви (например, суммарная стоимость всех комплектующих для сборки изделия).
  2.4 Перемещение поддерева (Reparenting): Изменение родителя у узла вместе со всеми его потомками.
  2.5 Проверка на цикличность: Предотвращение ситуаций, когда узел становится родителем для своего собственного предка.
  
3.	Для чего предназначен иерархический тип данных? 
  Для оптимизации хранения положения узла в дереве и упрощения навигации по нему.
  
4.	Перечислите известные вам свойства и методы иерархического типа данных.
  4.1 GetAncestor(n): Возвращает предка на уровне n.
  4.2 GetDescendant(child1, child2): Генерирует новое значение для вставки потомка между двумя существующими узлами.
  4.3 GetLevel(): Возвращает целое число, представляющее уровень узла в дереве.
  4.4 GetRoot(): Статический метод, возвращающий корневой элемент.
  4.5 IsDescendantOf(parent): Возвращает истину, если текущий узел является потомком указанного родителя.
  
5.	Поясните секции иерархических запросов в Oracle.
В Oracle иерархические запросы строятся с помощью конструкции SELECT ... FROM ... WHERE ..., дополненной специальными секциями:
  5.1 START WITH: Определяет условия для выбора «корневых» узлов, с которых начнется построение дерева.
  5.2 CONNECT BY: Определяет связь между родителем и потомком.
    Используется оператор PRIOR, который указывает на значение из «предыдущей» (родительской) строки.
    Например: CONNECT BY PRIOR ID = PARENT_ID означает «текущий PARENT_ID должен быть равен ID предыдущей строки».
  5.3 ORDER SIBLINGS BY: Позволяет сортировать узлы внутри одного уровня (между «братьями»), не нарушая при этом визуальную структуру самого дерева.

6.	Перечислите известные вам псевдофункции иерархических запросов в Oracle.
  6.1 LEVEL: Псевдостолбец, возвращающий номер уровня (1 для корня, 2 для детей и т.д.).
  6.2 CONNECT_BY_ISLEAF: Возвращает 1, если узел является «листом» (не имеет детей), и 0 в противном случае.
  6.3 CONNECT_BY_ISCYCLE: Возвращает 1, если текущий узел является причиной зацикливания (используется только вместе с NOCYCLE).
  6.4 SYS_CONNECT_BY_PATH(column, char): Формирует строку, представляющую полный путь от корня до текущего узла, разделяя значения символом char.
  6.5 CONNECT_BY_ROOT column: Позволяет получить значение указанного столбца из самой корневой строки текущей ветви.
