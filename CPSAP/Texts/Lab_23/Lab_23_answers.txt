1. Что такое хеш-функция?
Это функция, которая преобразует входной массив данных произвольной длины в выходную битовую строку фиксированной длины.

2. Что такое хеширование?
Это сам процесс преобразования данных с помощью хеш-функции.
В отличие от шифрования, хеширование является необратимым (односторонним) процессом. Это значит, что, имея на руках результат (хеш), невозможно математически восстановить исходный текст.

3. Перечислите алгоритмы хеширования.
SHA-1(устаревший), SHA-2(includes SHA-256, SHA-512), SHA-3(новейший), MD5 (Message Digest 5)

4. Поясните понятие «сеансовый ключ»
Это симметричный ключ(в симметричном шифровании данные, зашифрованные с помощью приватного ключа могут быть расшифрованы с помощью публичного и наоборот), 
который создается для защиты данных только в рамках одного конкретного сеанса связи между клиентом и сервером.

5. Поясните понятия «публичный» и «секретный» ключи?
Публичный (открытый) ключ: доступен любому желающему. Используется либо для зашифрования сообщения, либо для проверки цифровой подписи.
Секретный (приватный) ключ: хранится в строгом секрете у владельца. Используется для расшифрования данных или для создания цифровой подписи.

6. Поясните понятие «дайджест».
Дайджест сообщения (Message Digest) — это и есть результат работы хеш-функции (то есть сам хеш). 
Термин подчеркивает, что это «краткое содержание» или «отпечаток» большого объема данных в сжатом, фиксированном виде.

7. Для чего используется дайджест.
  7.1 Проверка целостности: если хеш файла совпадает с эталонным, значит, файл не был изменен при передаче.
  7.2 Создание цифровой подписи: подписывать огромный файл целиком долго и ресурсозатратно, поэтому подписывается только его дайджест.
  7.3 Хранение паролей: в базах данных хранят не сами пароли, а их дайджесты (хеши), чтобы при утечке базы данных злоумышленник не узнал реальные пароли.
  
8. Поясните принцип работы алгоритма Диффи-Хеллмана.
  8.1 Стороны договариваются об общих параметрах: большом простом числе p и основании g.
  8.2 Сервер выбирает секретное число a и вычисляет публичное значение A = g^a (modp)
  8.3 Клиент  выбирает секретное число b и вычисляет публичное значение B = g^b(modp)
  8.4 Они обмениваются A и B.
  8.5 Сервер вычисляет K = B^a(modp)
  8.6 Клиент вычисляет K = A^b(modp)
  8.7 Результат K у обоих одинаков g^ab(modp), хотя секреты a и b никогда не передавались по сети.
  
9. Поясните понятие «цифровая подпись», объясните принцип ее применения.    
Электронная цифровая подпись (ЭЦП) — это криптографический инструмент, подтверждающий авторство и неизменность документа.
Принцип применения:
  9.1 Создание подписи (От данных берется хеш (дайджест), зашифровывается секретным ключом отправителя и полученная подпись прикрепляется к документу)
  9.2 Проверка подписи (Получатель расшифровывает подпись с помощью публичного ключа отправителя и получает "исходный" хэш (дайджест), Получатель сам вычисляет дайджест полученного документа
   и если два дайджеста совпали — подпись верна: данные не менялись и присланы владельцем ключа.)
