1. In-Memory Database(IMDB) - это тип базы данных, который хранит данные непосредственно в оперативной памяти, что позволяет получить более бытсрый доступ и обработку данных в
сравнении с базами данных, распологающимися на диске

2.Хэш-Таблица - структура данных, которая хранит данные в формате "Ключ-Значение" и позволяющая быстро извлекать данные путём расчёта индекса с помощью хэш-функции

3. Каким образом обеспечивается персистентность  данных в СУБД Redis?
Имеется два механизма: RDB(Redis Data Base), он же Snapshotting и AOF(Append Only Files). Могут использоваться совместно, раздельно, а могут быть вообще отключены
RDB - Redis сохраняет снимок всего набора данных в бинарный файл, обеспечивается быстрое восстановление
AOF - Redis сохраняет все команды, выполненные над данными в файл и после перезапуска воспроизводит цепочку команд из этого файла для восстановления состояния(по сути логи)

4. Поясните назначение команд СУБД Redis:  set, get, del, getset.
set - сохранить строковое значение под заданным ключём(если такой ключ со значением уже есть - перезаписывает)
get - прочитать строковое значение, соответствующее переданному ключу
del - удалить указанные ключи из БД(один или несколько)
getset - атомарная операция. Возвращает старое значение ключа и устанавливает новое одновременно

5. Поясните назначение команд СУБД Redis:  incr, decr.
incr - атомарная операция. увеличивает целочисленное значение ключа на 1  возвращает новое значение после инкремента. Если ключ не существует - он считается равным 0 и incr вернёт 1
Если значение - не строка, являющаяся или содержащая целое число - incr вернёт ошибку.

decr - атомарная операция. Уменьшает целочисленное значение ключа на 1. возвращает новое значение после декремента. Если ключ не существует - он считается равным 0 и decr вернёт -1.
Если значение - не строка, являющаяся или содержащая целое число - decr вернёт ошибку.


6. Поясните назначение команд СУБД Redis:  mset, mget.

mset - утсанавливает одновременно несколько ключей со строковыми значениями. если ключам уже установлены значения - перезаписывает. Возвращает ОК при успешном завершении

mget - читает сразу несколько ключей в одном вызове. возвращает массив значений в порядке запрошенных ключей.


7. Поясните назначение команд СУБД Redis:  hset, hget.  

hset - установить одну или несколько пар поле->значение в хэш-структуре, хранящейся по ключу. Если ключа нет - создать новый

hget - получить значение конкретного поля в хэше по ключу

8. Поясните назначение команд СУБД Redis:  hmset,hmget.    
 hmset - установить сразу несколько пар поле-значение в хэше по ключу. Если клюа нет - создаст . Считается устаревшей командой
 hmget - получить значения сразу нескольких полей в хэше в одном вызове. вернёт массив в порядке запрошенных ключей
 
9. Поясните назначение команды СУБД Redis: exists.    
exists -  проверяет, существует ли один или несколько ключей в БД. Вернёт целое число, соответствующее количеству существующих ключей из переданных.
