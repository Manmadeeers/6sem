3. Что такое JSON RPC?
Легковесный RPC протокол, который использует JSON для кодирования вызовов и ответов
4. Опишите формат RPC-запроса.
{
  jsonrpc:"version",
  method:"method_name",
  params:"parameters for method call"
  id:"id"
}

5. Позиционные параметры запроса.
Параметры передаются как массив: "params": [arg1, arg2, arg3, ...].
Порядок элементов имеет значение и сопоставляется с позициями аргументов вызываемого метода.

6. Именованные параметры запроса.
Параметры передаются как объект: "params": { "x": 1, "y": 2 }.
Значения сопоставляются по именам ключей, что улучшает читаемость и гибкость.

7. Опишите формат RPC-ответа.
Успешный ответ:
jsonrpc: "2.0"
result: любое значение, вернувшееся методом
id: тот же идентификатор запроса

Ответ с ошибкой:
jsonrpc: "2.0"
error: объект с кодом и сообщением (и возможно data)
id: тот же id, что в запросе (или null, если его нет)

8. Что такое RPC-уведомление?
Это запрос без поля id. Трактуется как уведомление, и сервер по стандарту не обязан отправлять ответ

9. Что такое пакет RPC-запросов?
По сути Batch-запрос: массив отдельных RPC-запросов, которые должны быть обработаны вместе.
Формат: [req1, req2, req3, ...], где каждый req — объект JSON-RPC

Ответ — массив ответов: [{jsonrpc:"2.0", result: ..., id: ...}, ...], но учитывая уведомления, если в батче были только уведомления,
сервер может не возвращать тело (204 No Content) или вернуть пустой массив в некоторых реализациях.
